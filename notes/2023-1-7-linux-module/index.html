Linux内核模块

Linux提供了将部分功能单独编译，动态加载到内核的机制，这就是内核模块。

模块加载函数#

通过insmod或modprobe命令加载内核模块时，模块的加载函数会自动被内核执行，完成模块的相关初始化工作。
模块加载函数一般以__init标识声明，以module_init(函数名)的形式指定。

static int __init initialization_function(void)
{
    /*初始化代码*/   
}
module_init(initialization_function);

在linux中所有标识为__init的函数如果直接编译进内核，在链接时会放在.init.text区段，并在.initcall.init中保存一份函数指针，初始化时内核会通过函数指针调用这些函数。

模块卸载函数#

static void __exit cleanup_function(void)
{
    /*释放代码*/
}
module_exit(cleanup_function);

如果模块被直接编译进内核，则用__exit修饰的卸载函数会被省略，直接不链接到最后的镜像。因为模块被内置了，就不可能卸载它。

模块参数#

可以用module_param(参数名，参数类型，参数读/写权限)为模块定义一个参数。
通过insmod/modprobe 模块名.参数名=参数值来向模块传递参数。如果模块被内置,可以在bootargs里设置。

导出符号#

linux的/proc/kallsyms文件对应着内核符号表，模块可以通过使用如下宏导出符号到内核符号表中：

EXPORT_SYMBOL(符号名);
EXPORT_SYMBOL_GPL(符号名);

导出的符号可以被其他模块使用，只需要使用前声明一下即可。EXPORT_SYMBOL_GPL()只适用于GPL许可权的模块。

模块声明与描述#

MODULE_AUTHOR(author);
MODULE_DESCRIPTION(description);
MODULE_VERSION(version);
MODULE_DEVICE_TABLE(table);
MODULE_ALIAS(alias);

模块的使用计数#

Linux2.6以后内核提供了模块计数管理接口try_module_get(&module)和module_put(&module)，模块的使用计数一般不必由模块自身管理。
Linux2.6以后的内核为不同类型设备定义了struct module *owner域，用来指向管理此设备的模块。当开始使用某个设备时，内核使用try_module_get(dev->owner)去增加管理此设备的owner模块计数；反之，使用module_put(dev->owner)减少计数。这样，当设备在使用时，管理此设备的模块不能被卸载。
一般模块的计数管理由内核里更底层的代码（如总线驱动或此类设备共用的核心模块）来实现，从而简化设备驱动开发。

模块的编译#

模块的编译需要指定内核的build目录

obj-m = foo.o
KVERSION = $(shell uname -r)
all:
        make -C /lib/modules/$(KVERSION)/build M=$(PWD) modules
clean:
        make -C /lib/modules/$(KVERSION)/build M=$(PWD) clean
