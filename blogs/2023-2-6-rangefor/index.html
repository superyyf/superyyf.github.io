C++ range-based for loop与临时对象

February 6, 2023 · 1 min · Ye Yunfei
Table of Contents
  • range-based for loop 语法
  • range-expression 与临时变量

range-based for loop是从C++11开始出现的新语法，可以非常简洁的实现序列迭代。在实际使用中，发现了一个需要注意的细节，就是当range-expression表达式中出现临时对象时需要十分小心。

range-based for loop 语法#

attr(optional) for ( init-statement(optional) range-declaration : range-expression )
loop-statement

range-declaration是序列元素的声明，range-expression是表示某一个序列的表达式，这个序列可以是数组，有begin()、end()成员函数的可迭代对象，或者初始化队列。 上述的语法可以等效成下面的一组语句：

{

    auto && __range = range-expression ;
    for (auto __begin = begin-expr, __end = end-expr; __begin != __end; ++__begin)
    {

        range-declaration = *__begin;
        loop-statement

    }

}

range-expression 与临时变量#

尝试编译运行下面的代码

#include <iostream>
#include <memory>
#include <string>
#include <vector>

struct xxx {
  std::vector<std::string> vec;
};

std::shared_ptr<struct xxx> f() {
  auto p = std::make_shared<struct xxx>();
  p->vec.push_back("aaa");
  p->vec.push_back("bbb");
  return p;
}

int main() {
  for (auto str : f()->vec) {
    std::cout << str;
  }
  std::cout << std::endl;
}

一眼看上去并没有毛病，但是仔细分析for(auto str : f()->vec)这一句相当于

auto&& expression = f()->vec;
for(auto p = expression.begin(); p != expression.end(); ++p){
    ...
}

f()的返回值是个临时值，当第一个语句结束时返回的智能指针的生命周期就结束了，它所指向的结构体随之析构。然后下面语句中的迭代器指针失效，造成段错误。
